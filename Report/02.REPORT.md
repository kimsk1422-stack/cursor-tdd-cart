## 1. 요약

이번 세션은 MomTest 방식 Product Discovery 에이전트 역할로 장바구니 할인 계산기 도메인 규칙을 분석했다. 첫 턴에서는 요구사항 미입력 상태에서 코드베이스에 존재하는 `INV-001`(소계 = 단가 × 수량)만 근거로 계약을 제한하고 MomTest 질문·OOS 표를 작성했다. 두 번째 턴에서 사용자가 인터뷰 메모·주문 사례·엑셀 검산·CS 클레임을 제공했고, 이를 바탕으로 **INV-001~INV-005**, **E-001~E-002**, **OOS-001~OOS-009** 계약표를 확정했다. `src/`·`tests/` 코드는 변경하지 않았다.

## 2. 핵심 결정·산출물

- **번호 규칙**: 기존 `01` 존재 → 이번 Export는 **02**
- **세션 주제**: MomTest 인터뷰 메모 분석 → 테스트 가능한 계약 ID 추출
- **확정 Entity 계약**
  - `INV-001`: 라인 소계 `price * quantity`
  - `INV-002`: 장바구니 소계 = 라인 합
  - `INV-003`: 소계 `> 50000` → `round(subtotal * 0.9)`, 이하 → 그대로
  - `INV-004`: VIP — 문턱 할인 후 `round(amount * 0.95)`, 순서 고정
  - `INV-005`: 유효 입력 시 결제 금액 ≤ 소계
  - `E-001`: `quantity < 0` → `ValueError` (0원 조용 계산 금지)
- **확정 Boundary 계약**
  - `E-002`: 빈 `items` 제출 시 HTTP 500 금지 (검증 오류)
- **명시적 OOS**: 쿠폰·배송비, 소계 정확히 50,000원, VIP+소계≤50,000, 음수 단가·수량 0 등
- **생성 파일**

  - `Prompting/02.Export-Transcript.md` — 대화 전문 Export
